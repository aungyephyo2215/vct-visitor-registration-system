# Visitor Registration System v1.2.0

Overview

This release adds in-app notifications, dashboard invitation stats, and solidifies the development toolchain with Prettier, Husky, lint-staged, and Vitest unit tests.

---

New Features

Notification Module

- In-app notification system
- Bell icon with unread badge in header
- Dropdown list with relative timestamps
- Mark single notification as read
- Mark all notifications as read
- Automatic notification delivery for 7 events:
  - INVITATION_CREATED — all approvers notified
  - INVITATION_APPROVED — inviter notified
  - INVITATION_REJECTED — inviter notified with reason
  - QR_GENERATED — inviter notified
  - VISITOR_VERIFIED — host notified
  - CHECKED_IN — host notified
  - CHECKED_OUT — host notified

---

Notification APIs

- `GET /api/v1/notifications` — list with pagination, unread-first ordering
- `GET /api/v1/notifications/unread-count` — fast badge counter
- `PATCH /api/v1/notifications/:id/read` — mark single read
- `PATCH /api/v1/notifications/read-all` — mark all read

Technical Details

Migration

```
20260617154134_add_notifications
```

Creates `notifications` table with `NotificationType` and `NotificationResourceType` enums.

Library Architecture

```
src/lib/notifications/
├── templates.ts     — title/message per notification type (pure)
├── builder.ts       — flat data constructors (pure)
├── recipients.ts    — who-gets-notified logic (pure)
└── service.ts       — DB writes, fire-and-forget wrappers
```

Fire-and-forget design — notification failure never breaks the parent operation (invitation creation, check-in, etc.).
