# Visitor Registration System v1.3.0

Overview

This release introduces QR Email Delivery, allowing invitations to automatically send QR codes to visitors via email.

The system now supports SMTP email delivery, manual resend, public QR access, and hosted QR image rendering.

This release also includes important security improvements and delivery reliability enhancements.

Security Improvements

Fixed Invitation PATCH Status Escalation

Removed status updates from:

PATCH /api/v1/invitations/:id

Status transitions are now only allowed via:

- POST /approve
- POST /reject

with role-based authorization.

⸻

QR Email Token Cleanup

Prevent orphan email access tokens from being stored when email delivery is skipped.
